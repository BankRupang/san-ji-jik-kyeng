# CI 파이프라인

## 전체 흐름

`main` 또는 `dev` 브랜치를 대상으로 PR이 열리면 자동으로 실행됩니다.

```
PR 생성 / 커밋 push
        │
        └─── [ci.yml] 자동 실행
                   Gradle 캐시 복원
                   -> 빌드 (테스트 제외)
                   -> 테스트
```

CI를 통과한 코드만 merge될 수 있고, merge되면 CD가 자동으로 배포를 시작합니다.

---

## 트리거

| 조건 | 동작 |
|---|---|
| `main` 브랜치 대상 PR | 자동 실행 |
| `dev` 브랜치 대상 PR | 자동 실행 |

---

## 단계별 설명

**1. Gradle 캐시 복원**

`build.gradle`과 `gradle-wrapper.properties`의 해시값을 키로 캐시를 저장하고 복원합니다. 의존성이 바뀌지 않은 경우 다운로드를 생략해 실행 시간을 줄입니다.

**2. 빌드**

`./gradlew build -x test --parallel`로 전체 서비스를 빌드합니다. `-x test`로 테스트를 건너뛰고 컴파일과 JAR 생성만 수행합니다. 빌드 실패와 테스트 실패를 단계별로 분리해서 원인을 바로 알 수 있습니다.

**3. 테스트**

`./gradlew test --parallel`로 전체 서비스 테스트를 실행합니다.
